  6. (45 marks) Not the Smallest, Not the Largest, the Middle

    (We did this in class this year, but it was on last year's final exam. I include it here to give you an idea of the difficulty of the longer problems on the exam).

    7. The median value in a list is the value that would be in the middle if the list was sorted, i.e. the same number of values are smaller than the median as are larger than it*.

    1. Start with the simplest thing that could possibly work. Write a function that returns the median of a list of positive numbers using the following approach.

      consider each value in the list (until the median is found)
    count how many values in the list are smaller than it
    and how many values are larger
    based on the counts above decide if it is the median

      If the test is carefully written this can work. Show the test cases you use in verifying your code.

    2. What order is this algorithm?
    3. While correct, the approach in a) above is inefficient. For example, if we�re working with the list [31, 18, 72, 91, 84, 53, 25] we will discover in the first step that 31 is not the median because there are 3 numbers less than or equal to it, but 5 numbers greater than or equal to it. From this we know not only that 31 is not the median, but that the median must be some number larger than 31. This means that when we get to the value 18 we can skip it, i.e. not bother counting how many values are larger or smaller than it, because we know it cannot be the median. Similarly once we discover that 72 is not the median we can skip 91 and 84. Modify your code from a) based on this insight.

    *Fine print: Actually it is the number of values smaller than or equal to it, and larger than or equal to it. Furthermore, if there is an even number of values in the list the median may not be easily defined, e.g. what is the median value in the list [1, 2, 3, 4]? In this case your code may return -1.
